# 🚀 Exit Code 0 - Algorithm Track

**Exit Code 0 - Algorithm Track**은 코딩테스트 문제 풀이 루틴을 만들고, 오답노트를 통해 풀이 과정을 복기하며, 제한 시간 안에서 문제를 해결하는 실전 감각을 기르는 것을 목표로 합니다.

단순히 많은 문제를 푸는 것보다,
**꾸준히 기록하고 → 틀린 이유를 분석하고 → 다음 풀이에 반영하는 것**을 중요하게 생각합니다.

---

## 📌 스터디 개요

### 진행 기간

* 약 8~9주
* TF 진행 상황 및 참여자 성과에 따라 추가 운영 가능
* 세부 일정은 OT 이후 확정

### 제출 기한

* 매주 일요일 자정까지 제출

### 진행 방식

* 매주 주차별 알고리즘 주제 확인
* 개인별 목표 문제 수 설정
* 문제 풀이 및 오답노트 작성
* GitHub Pull Request로 제출
* 스터디 시간에 풀이 과정, 어려웠던 점, 개선 포인트 공유
* 2주에 한 번 격주로 라이브 코딩테스트 진행

## 🧭 진행 방식

본 TF의 핵심은 **문제 풀이 인증**과 **오답노트 작성**입니다.

매주 정해진 알고리즘 주제를 기반으로 간단한 개념 공유를 진행하고, 문제 풀이는 각자 본인의 목표와 난이도에 맞춰 진행합니다.

문제는 해당 주차 주제와 관련된 문제를 우선 권장하지만, 개인별 취업 목표나 부족한 유형에 따라 자유롭게 선택할 수 있습니다.

### 매주 진행 흐름

1. 주차별 알고리즘 주제 확인
2. 각자 목표 문제 수 설정
3. 일주일 동안 문제 풀이
4. GitHub에 풀이 인증 및 오답노트 제출
5. 스터디 시간에 어려웠던 문제, 풀이 방식, 개선 포인트 공유
6. 격주로 라이브 코딩테스트 진행

---

## 🗓️ 주차별 주제

기본 유형은 초반에 빠르게 점검하고, DFS/BFS, 완전탐색/시뮬레이션, DP, 그래프처럼 시간이 오래 걸리는 유형에 더 많은 비중을 둡니다.

| 주차      | 주제                          | 주요 내용                                          |
| ------- | --------------------------- | ---------------------------------------------- |
| Week 01 | OT / 구현 / 문자열 / 배열          | 스터디 규칙, GitHub 사용 방식, 기본 입출력, 문자열, 배열, 구현      |
| Week 02 | Stack / Queue / Hash / 정렬   | 기본 자료구조, 해시 활용, 정렬                             |
| Week 03 | Greedy / 이분탐색 / 투 포인터 / 누적합 | 정렬 기반 사고, 탐색 범위 줄이기, 효율화                       |
| Week 04 | DFS / BFS 1                 | 기본 탐색, visited 처리, 그래프/격자 탐색                   |
| Week 05 | DFS / BFS 2                 | 최단거리, 상태 탐색, 다중 조건 탐색                          |
| Week 06 | 완전탐색 / 백트래킹 / 시뮬레이션 1       | 순열, 조합, 부분집합, 브루트포스                            |
| Week 07 | 완전탐색 / 시뮬레이션 2              | 구현량이 많은 문제, 조건 처리, 디버깅                         |
| Week 08 | DP 기초 / 심화                  | 1차원 DP, 2차원 DP, 점화식 도출                         |
| Week 09 | 그래프 / 최단거리 / 종합 실전          | Union-Find, Dijkstra, Floyd-Warshall, 유형 혼합 문제 |

> 스터디 진행 상황에 따라 Week 08~09는 DP, 그래프, 실전 모의고사 중심으로 조정될 수 있습니다.

## 💻 라이브 코딩테스트

라이브 코딩테스트는 **2주에 한 번 격주로 진행**합니다.

실제 코딩테스트와 비슷한 환경에서 제한 시간 안에 문제를 해결하는 연습을 목표로 하며, 대면 진행을 원칙으로 합니다.

### 기본 방식

* 정해진 시간과 장소에 모여 동시에 시작
* 제한 시간 내 각자 문제 풀이
* 종료 후 풀이 여부와 접근 방식 공유
* 풀지 못한 문제도 시도한 방식과 막힌 지점 공유
* 이후 오답노트 작성

### 문제 구성 예시

| 번호 | 유형           |
| -- | ------------ |
| 1번 | 기본 구현 / 자료구조 |
| 2번 | 주차 핵심 유형     |
| 3번 | 응용 문제        |
| 4번 | SQL          |

> 문제 수와 난이도는 참여자들의 진행 상황에 따라 조정합니다.
> 대면 진행이 어려운 주차에는 운영진 판단에 따라 Discord 기반 비대면 라이브 코딩테스트로 대체할 수 있습니다.

매주 정해진 제출일까지 아래 내용을 제출합니다.

### 필수 제출 항목

* 문제 풀이 인증
* 오답노트
* 다음 주 목표 문제 수

참여자는 매주 **최소 3문제 이상**을 인증해야 합니다.

목표 문제 수를 채우지 못했더라도 제출은 반드시 진행합니다.
못 푼 문제도 막힌 지점과 시도한 방식을 작성하면 오답노트로 인정합니다.

## 📝 오답노트 작성 원칙

오답노트는 AI 답변이나 블로그 풀이를 그대로 복사하지 않고, **본인의 언어로 직접 작성하는 것**을 원칙으로 합니다.

AI를 활용해 개념을 확인하거나 풀이를 비교하는 것은 가능하지만, 최종 제출물은 반드시 본인이 이해한 방식으로 정리해야 합니다.

### 오답노트에 포함할 내용

```markdown

- 실제 풀이 문제 수:

## 푼 문제 목록

| 번호 | 문제 이름 | 난이도 | 링크 | 풀이 여부 |
| --- | --- | --- | --- | --- |
| 1 |  |  |  |  |
| 2 |  |  |  |  |
| 3 |  |  |  |  |

---
```

## 💰 보증금 및 벌금 안내

본 TF는 꾸준한 참여와 과제 제출을 위해 보증금 제도로 운영합니다.

### 보증금

* 20,000원

### 벌금

* 해당 주차 참여 기준 미충족 시 2,000원 차감
* 한 주 최대 차감 금액은 2,000원

### 환급 방식

* TF 종료 후 벌금을 제외한 나머지 금액 환급

### 벌금 기준

과제는 **문제 풀이 인증**과 **오답노트 제출**을 모두 포함합니다.

다음 중 하나에 해당할 경우 해당 주차 벌금이 차감됩니다.

* 문제 풀이 인증 미제출
* 오답노트 미제출
* 최소 3문제 미만 제출
* 무단 결석

단, 한 주에 과제 미제출과 무단 결석이 동시에 발생해도 최대 2,000원만 차감됩니다.

사전에 결석 사유를 공유한 경우에는 무단 결석으로 보지 않습니다.
다만 결석 여부와 관계없이 해당 주차 과제를 제출하지 않으면 벌금이 차감됩니다.

환급되지 않은 벌금은 TF 종료 후 참여자들과 상의하여 회식비, 기프티콘, 현금 재분배 중 하나의 방식으로 사용합니다.

## 📌 운영 규칙

1. 매주 최소 3문제 이상을 풀이합니다.
2. 문제는 본인 난이도와 목표에 맞춰 자유롭게 선택할 수 있습니다.
3. 가능하면 해당 주차 주제와 관련된 문제를 최소 1문제 이상 포함합니다.
4. 문제 풀이 인증과 오답노트를 제출합니다.
5. 목표 문제 수를 달성하지 못했더라도 제출은 반드시 진행합니다.
6. 못 푼 문제도 시도한 방식과 막힌 지점을 작성하면 오답노트로 인정합니다.
7. 오답노트는 AI나 블로그 풀이를 그대로 복사하지 않고 본인의 언어로 작성합니다.
8. 질문할 때는 막힌 지점과 시도한 방식을 함께 공유합니다.
9. 라이브 코딩테스트에서는 풀이 여부보다 접근 과정과 복기를 중요하게 봅니다.
10. 스터디의 핵심은 완벽한 풀이가 아니라 꾸준한 문제 풀이 루틴 형성입니다.

## ⚙️ LeetHub / BaekjoonHub 자동 커밋 안내

LeetHub 또는 BaekjoonHub를 사용하는 경우, 자동 커밋된 문제 풀이가 GitHub Actions를 통해 플랫폼별 디렉토리로 정리될 수 있습니다.

자동 정리 워크플로우는 `.github/workflows` 디렉토리에 별도로 관리합니다.

개인 환경 설정 시 아래 정보는 본인의 GitHub 계정에 맞게 수정합니다.

```bash
git config --global user.name "본인 GitHub 아이디"
git config --global user.email "본인 이메일"
```

## 🎯 우리의 목표

```bash
No Record → Real Error
Wrong Answer → Review
Time Limit Exceeded → Optimize
Accepted → Exit Code 0
```

완벽한 사람보다, 꾸준히 기록하고 개선하는 사람이 성장한다고 생각합니다.

Algorithm Track의 목표는 단순히 문제 수를 채우는 것이 아니라,
매주 조금씩 더 빠르고 정확하게 문제를 해결할 수 있는 사람이 되는 것입니다.

# 문제 1

## 문제 정보

* **문제명:** 33. Search in Rotated Sorted Array
* **문제 링크:** https://leetcode.com/problems/search-in-rotated-sorted-array/description/
* **사용한 알고리즘 / 자료구조:** 이분 탐색(Binary Search)

---

## 처음 접근 방식

처음에는 일반적인 이분 탐색을 그대로 적용하려고 했다.

배열의 중간값(`mid`)과 `target`을 비교하여,

* `nums[mid] > target`이면 왼쪽 범위를 탐색하고,
* `nums[mid] < target`이면 오른쪽 범위를 탐색하는 방식으로 접근하였다.

하지만 문제의 배열은 **회전된 정렬 배열(Rotated Sorted Array)** 이기 때문에 단순히 `mid`와 `target`만 비교해서는 탐색 방향을 결정할 수 없었다.

예를 들어,

```text
[4, 5, 6, 7, 0, 1, 2]
```

에서 `target = 0`인 경우,

* `mid = 7`
* `7 > 0`

이지만 실제 정답은 오른쪽에 존재한다.

즉, 일반 이분 탐색의 전제가 되는 **"배열 전체가 정렬되어 있다."** 는 조건이 성립하지 않는다는 점을 처음에는 제대로 고려하지 못했다.

---

## 풀이 방법

이 문제의 핵심은 **회전된 배열이라도 항상 한쪽 구간은 정렬되어 있다**는 성질을 이용하는 것이다.

### 1. mid가 target인지 확인한다.

```java
if (nums[mid] == target)
```

정답이라면 즉시 해당 인덱스를 반환한다.

### 2. 왼쪽 구간이 정렬되어 있는지 확인한다.

```java
nums[left] <= nums[mid]
```

이 조건이 참이면

```
[left ~ mid]
```

구간이 오름차순으로 정렬되어 있다는 의미이다.

### 3. target이 정렬된 왼쪽 구간 안에 존재하는지 확인한다.

```
nums[left] <= target < nums[mid]
```

* 포함된다면 오른쪽 구간을 버리고

  ```
  right = mid - 1
  ```
* 포함되지 않는다면 왼쪽 구간을 버리고

  ```
  left = mid + 1
  ```

### 4. 오른쪽 구간이 정렬되어 있는 경우

왼쪽이 정렬되어 있지 않다면 오른쪽 구간이 정렬되어 있다.

```
[mid ~ right]
```

이 구간 안에 target이 존재하는지 확인한 후,

* 포함되면

  ```
  left = mid + 1
  ```
* 포함되지 않으면

  ```
  right = mid - 1
  ```

를 수행한다.

이 과정을 반복하면 탐색 범위를 절반씩 줄이며 O(log N) 안에 target을 찾을 수 있다.

---

## 어려웠던 지점

가장 어려웠던 부분은 **일반 이분 탐색을 그대로 적용하려고 했던 점**이었다.

처음에는

```java
if(nums[mid] > target)
```

이면 왼쪽,

```java
else
```

이면 오른쪽으로 이동하는 방식으로 구현했는데,

회전된 배열에서는 이러한 비교만으로는 target의 위치를 판단할 수 없었다.

또한 이분 탐색 문제를 풀면서 항상 어려웠던 부분은

* 어떤 기준으로 탐색 범위를 줄여야 하는지
* 마지막에 무엇을 반환해야 하는지

를 정확하게 판단하는 것이었다.

---

## 해결 방법

회전된 배열에서는 **mid와 target만 비교하는 것이 아니라, 먼저 어느 구간이 정렬되어 있는지를 확인해야 한다**는 점을 이해하면서 해결할 수 있었다.

과정은 다음과 같다.

1. 왼쪽이 정렬되어 있는가?
2. 아니라면 오른쪽이 정렬되어 있는가?
3. target이 정렬된 구간 안에 존재하는가?
4. 존재하면 그 구간만 남기고, 아니라면 반대 구간만 남긴다.

즉,

> **정렬된 구간을 먼저 찾고, target이 그 구간 안에 있는지 판단하여 탐색 범위를 절반으로 줄이는 것**이 문제의 핵심이었다.

## 시간복잡도 / 공간복잡도

* **시간복잡도:** O(log N)
* **공간복잡도:** O(1)

---

## 새롭게 알게 된 점

이번 문제를 통해 **회전된 정렬 배열에서의 이분 탐색은 일반 이분 탐색과 탐색 기준 자체가 다르다**는 것을 배웠다.

이후 비슷한 문제를 만나면 다음 순서대로 접근할 예정이다.

1. 이분 탐색 적용이 가능한 문제인지 확인한다.
2. 탐색 범위를 절반으로 줄일 수 있는 기준이 무엇인지 찾는다.
3. 일반 정렬 배열인지, 회전 배열인지 등 추가 조건을 확인한다.
4. 정렬된 구간을 먼저 찾고, target이 그 구간 안에 있는지를 판단하여 범위를 줄인다.
5. 탐색 종료 조건과 최종 반환값(left, right, mid 등)을 먼저 명확히 정한 뒤 구현한다.

특히 이번 문제를 통해 **이분 탐색은 단순히 `mid`를 계산하는 알고리즘이 아니라, "어떤 기준으로 탐색 범위를 절반으로 줄일 것인가"를 설계하는 알고리즘**이라는 점을 다시 한 번 느낄 수 있었다.
